# Cabin 项目设计规范

## 1. 视觉设计规范

### 1.1 Windows 95 风格指南
- **色彩系统**：使用经典的 Windows 95 色彩方案
  - 桌面背景：深绿色/青色 (`teal`)
  - 窗口标题栏：深蓝色
  - 按钮和控件：灰色系配合 3D 效果
- **字体**：使用系统默认字体，保持复古感
- **图标风格**：16x16 像素风格图标，符合 Windows 95 时代特征

### 1.2 按钮和控件规范
- **正常状态**：
  - 上边框和左边框：白色 (#ffffff) 高光
  - 下边框和右边框：黑色 (#000000) 阴影
- **按下状态**：
  - 上边框和左边框：深灰色 (#808080) 阴影  
  - 下边框和右边框：白色 (#ffffff) 高光
- **禁用状态**：灰色调，降低对比度

### 1.3 窗口系统规范
- **窗口组成**：
  - 标题栏（包含窗口标题和控制按钮）
  - 客户区（主要内容区域）
  - 状态栏（可选）
- **窗口操作**：
  - 拖拽移动：点击标题栏拖动
  - 最小化：点击最小化按钮，窗口缩小到任务栏
  - 关闭：点击关闭按钮，窗口从桌面移除
- **窗口层级**：点击窗口使其获得焦点，置于顶层

### 1.4 任务栏规范
- **位置**：固定在屏幕底部
- **高度**：28px（符合 Windows 95 标准）
- **组成元素**：
  - 开始按钮（左侧）
  - 活动窗口指示器（中间）
  - 系统托盘（右侧，预留）

## 2. 交互设计规范

### 2.1 桌面交互
- **双击打开**：双击桌面图标打开对应应用
- **右键菜单**：桌面右键显示上下文菜单
- **图标排列**：自动对齐网格，从左到右、从上到下排列

### 2.2 文件系统交互
- **文件夹导航**：双击文件夹进入，地址栏显示当前路径
- **文件打开**：双击文件使用关联应用打开
- **返回上级**：使用工具栏的向上箭头或地址栏导航

### 2.3 应用交互
- **多实例支持**：非唯一应用可同时打开多个实例
- **唯一实例**：如我的电脑、博客等应用只允许一个实例
- **窗口管理**：任务栏显示所有打开的窗口，支持切换

### 2.4 可移动磁盘交互
- **挂载提示**：显示"插入可移动磁盘"占位符
- **文件选择**：用户可选择本地文件进行虚拟挂载
- **临时存储**：挂载的文件仅在当前会话有效

## 3. 信息架构

### 3.1 目录结构设计
```
桌面/
├── 我的电脑/
│   ├── C:/
│   │   ├── Windows/ (系统文件夹)
│   │   ├── Program Files/
│   │   └── Users/
│   │       ├── Guest/
│   │       └── Admin/
│   │           ├── Desktop/
│   │           ├── Documents/
│   │           └── Downloads/
│   └── D:/
├── 回收站/
├── 我的文档/
│   ├── README.txt
│   └── 便签.txt
├── 我的图片/
├── 我的博客/
│   ├── Hello World.txt
│   ├── 技术笔记.txt
│   ├── 生活日记.txt
│   └── Win95风格网站.txt
└── 可移动磁盘/
    └── 插入可移动磁盘
```

### 3.2 应用映射关系
| 文件类型 | 关联应用 | 图标 |
|---------|---------|------|
| .txt, .md | 记事本 | notepad |
| 文件夹 | 文件资源管理器 | folder |
| 根目录 | 我的电脑 | my-computer |
| 博客目录 | 我的博客 | blog |

### 3.3 内容组织
- **博客文章**：按时间倒序排列，支持分类标签
- **文档页面**：按功能分类，便于快速查找
- **系统文件**：隐藏系统文件夹（Windows），普通用户不可见

## 4. 响应式设计

### 4.1 桌面优先
- 主要针对桌面设备优化
- 最小宽度：800px
- 推荐分辨率：1024x768 及以上

### 4.2 移动端适配
- **基本可用**：确保核心功能在移动端可访问
- **触摸优化**：增大点击区域，适应手指操作
- **布局调整**：任务栏在小屏幕上可能需要垂直排列

### 4.3 缩放支持
- 支持浏览器缩放
- 窗口尺寸随容器调整
- 图标和文字保持清晰度

## 5. 动画和过渡

### 5.1 窗口动画
- **打开效果**：窗口从中心淡入并缩放
- **关闭效果**：窗口淡出并缩小
- **最小化**：窗口缩小到任务栏位置

### 5.2 交互反馈
- **按钮点击**：立即显示按下状态
- **文件操作**：显示加载指示器
- **错误提示**：使用 Windows 95 风格对话框

### 5.3 性能考虑
- 动画时长：200-300ms，避免过长延迟
- 禁用选项：允许用户关闭动画以提升性能
- 流畅性：确保 60fps 的动画流畅度

## 6. 可访问性

### 6.1 键盘导航
- **Tab 键**：在可聚焦元素间切换
- **Enter 键**：激活当前选中项
- **方向键**：在列表和网格中导航
- **Alt+F4**：关闭当前窗口

### 6.2 屏幕阅读器支持
- **语义化 HTML**：使用正确的 ARIA 属性
- **焦点管理**：确保键盘焦点可见
- **状态通知**：窗口状态变化时通知用户

### 6.3 对比度要求
- **文本对比度**：至少 4.5:1
- **控件对比度**：确保可识别性
- **颜色独立**：不依赖颜色传达信息

## 7. 设计资产

### 7.1 图标库
- 使用 [@react95/icons](https://www.npmjs.com/package/@react95/icons)
- 自定义图标遵循 16x16 像素规范
- 图标命名与应用 ID 保持一致

### 7.2 色彩变量
```css
:root {
  --win95-bg: #008080; /* 青色背景 */
  --win95-border-light: #ffffff; /* 高光边框 */
  --win95-border-dark: #000000; /* 阴影边框 */
  --win95-button-pressed-light: #808080; /* 按下状态高光 */
}
```

### 7.3 字体栈
```css
font-family: 'MS Sans Serif', 'Tahoma', 'Arial', sans-serif;
```

## 8. 设计验证

### 8.1 视觉一致性检查
- [ ] 所有按钮使用统一的 3D 样式
- [ ] 窗口标题栏颜色一致
- [ ] 图标风格统一
- [ ] 字体大小和行高一致

### 8.2 交互一致性检查
- [ ] 双击行为在所有地方一致
- [ ] 右键菜单内容相关
- [ ] 窗口操作逻辑统一
- [ ] 错误处理方式一致

### 8.3 用户测试要点
- [ ] 新用户能否快速理解操作方式
- [ ] 常用功能是否容易找到
- [ ] 是否有混淆或困惑的交互点
- [ ] 怀旧体验是否真实还原
