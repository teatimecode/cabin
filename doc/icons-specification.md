# Cabin 项目图标规范

## 1. 图标系统概述

### 1.1 设计原则
- **复古风格**：严格遵循 Windows 95 时代的 16x16 像素图标风格
- **功能导向**：图标设计以功能识别为首要目标
- **一致性**：所有图标保持统一的视觉语言和设计规范
- **可扩展性**：支持未来新增图标的无缝集成

### 1.2 技术实现
- **图标库**：使用 [@react95/icons](https://www.npmjs.com/package/@react95/icons) 作为基础图标库
- **自定义图标**：通过 SVG 实现项目特定的图标需求
- **图标映射**：通过字符串名称映射到具体图标组件
- **响应式**：图标在不同缩放级别下保持清晰度

## 3. 图标使用规范

### 3.1 图标尺寸规范
- **桌面图标**：32x32 像素（显示尺寸），源文件 16x16 像素
- **任务栏图标**：16x16 像素
- **窗口内图标**：16x16 像素
- **工具栏图标**：16x16 像素

### 3.2 图标颜色规范
- **主色调**：使用 Windows 95 经典配色方案
- **背景透明**：所有图标使用透明背景
- **色彩限制**：每个图标不超过 16 色，符合 90 年代显示技术限制
- **对比度**：确保在各种背景下都清晰可见

### 3.3 图标命名规范
- **小写连字符**：使用 `kebab-case` 命名（如 `my-computer`）
- **语义化**：名称应准确描述图标功能
- **避免缩写**：使用完整单词，提高可读性
- **一致性**：同类图标使用相同前缀（如 `drive-*`）

## 4. 图标实现方式

### 4.1 基础图标组件
```typescript
// 图标获取函数
export function getIcon(iconName: string): React.ComponentType {
  const iconMap: Record<string, React.ComponentType> = {
    'my-computer': MyComputerIcon,
    'recycle-bin': RecycleBinIcon,
    'recycle-full': RecycleFullIcon,
    'folder': FolderIcon,
    'drive': DriveIcon,
    'drive-removable': RemovableDriveIcon,
    'explorer': ExplorerIcon,
    'notepad': NotepadIcon,
    'blog': BlogIcon,
    'up-arrow': UpArrowIcon,
    'left-arrow': LeftArrowIcon,
    // ... 其他图标
  };
  
  return iconMap[iconName] || DefaultFolderIcon;
}
```

### 4.2 图标注册机制
- **静态注册**：在构建时确定所有可用图标
- **动态扩展**：支持运行时添加新图标（通过可移动磁盘功能）
- **默认回退**：未找到图标时使用通用文件夹图标

### 4.3 图标缓存策略
- **组件级缓存**：使用 React.memo 优化图标渲染性能
- **名称映射缓存**：缓存图标名称到组件的映射关系
- **懒加载**：大型图标集支持按需加载

## 5. 自定义图标开发

### 5.1 SVG 规范
- **尺寸**：16x16 viewBox
- **路径优化**：使用最少的路径点数
- **样式内联**：所有样式属性直接写在 SVG 元素上
- **无障碍**：添加适当的 aria-label 属性

### 5.2 开发流程
1. **设计草图**：在像素网格上绘制图标
2. **SVG 导出**：导出为 SVG 格式
3. **React 组件转换**：将 SVG 转换为 React 组件
4. **注册到图标系统**：在 `getIcon` 函数中添加映射
5. **测试验证**：在不同场景下测试图标显示效果

### 5.3 质量检查清单
- [ ] 尺寸符合 16x16 规范
- [ ] 风格与现有图标一致
- [ ] 在深色和浅色背景下都清晰可见
- [ ] 文件大小合理（< 2KB）
- [ ] 无障碍属性完整

## 6. 图标维护

### 6.1 版本管理
- **图标版本**：每个图标变更都记录版本信息
- **向后兼容**：保留旧图标名称的映射关系
- **废弃流程**：标记废弃图标并提供迁移指南

### 6.2 性能监控
- **加载时间**：监控图标加载对页面性能的影响
- **内存占用**：确保图标组件不会造成内存泄漏
- **渲染性能**：优化频繁使用的图标渲染效率

### 6.3 用户反馈
- **图标识别度**：收集用户对图标含义的理解
- **视觉疲劳**：评估长时间使用后的视觉舒适度
- **文化适应性**：确保图标在不同文化背景下都能正确理解

## 7. 扩展场景

### 7.1 主题支持
- **图标主题**：支持不同视觉主题的图标变体
- **动态切换**：运行时切换图标主题
- **用户自定义**：允许用户上传自定义图标

### 7.2 动画图标
- **状态动画**：回收站满/空状态的切换动画
- **加载动画**：文件操作时的旋转等待图标
- **交互反馈**：点击时的微动效

### 7.3 高分辨率支持
- **Retina 显示**：提供 2x、3x 分辨率的图标版本
- **矢量缩放**：确保 SVG 图标在任意分辨率下都清晰
- **性能平衡**：在质量和性能之间找到最佳平衡点
