## 设计目标

设计一个完整的窗口管理器交互流程，支持：
- 打开/关闭窗口
- 最小化/最大化窗口
- 窗口拖拽和缩放
- 多窗口层级管理
- 任务栏集成

---

## 用户流程

### 打开窗口流程

```
用户双击桌面图标
    ↓
WindowManager 创建新窗口实例
    ↓
分配唯一 ID 和初始位置
    ↓
播放打开动画 (150ms)
    ↓
窗口显示在桌面
    ↓
自动获得焦点并置顶
    ↓
任务栏添加对应按钮
```

### 切换窗口流程

```
用户点击非激活窗口
    ↓
WindowManager 接收焦点事件
    ↓
计算新的 zIndex (max + 1)
    ↓
更新窗口层级
    ↓
更新任务栏按钮状态
    ↓
原激活窗口变为非激活态
```

## 交互细节规范

### 窗口拖拽

| 属性 | 值 | 说明 |
|------|-----|------|
| 触发区域 | 标题栏 | 整个标题栏可拖拽 |
| 延迟时间 | 50ms | 按下后延迟开始拖拽 |
| 移动阈值 | 5px | 超过 5px 才算拖拽 |
| 边界限制 | 桌面区域 | 不能完全拖出可视区域 |
| 吸附距离 | 20px | 接近边缘 20px 内自动吸附 |

### 窗口层级规则

```javascript
// z-index 管理策略
const zIndexRules = {
  desktop: 0,      // 桌面层
  taskbar: 1000,   // 任务栏固定在最上层
  windowBase: 100, // 窗口基础层级
  windowMax: 999,  // 窗口最大层级
};

// 新窗口/激活窗口的 zIndex 计算
function calculateZIndex(currentWindows) {
  const maxZ = Math.max(...currentWindows.map(w => w.zIndex), 0);
  return Math.min(maxZ + 1, zIndexRules.windowMax);
}
```

### 窗口尺寸规范

| 尺寸类型 | 宽度 | 高度 | 说明 |
|----------|------|------|------|
| 最小尺寸 | 300px | 200px | 窗口允许的最小大小 |
| 默认尺寸 | 640px | 480px | 新建窗口的默认大小 |
| 最大化 | 100% | 100% | 减去任务栏高度 |
| 初始位置 | 居中偏上 | -50px | 每次偏移避免完全遮挡 |

## 任务栏集成

### 按钮状态

| 窗口状态 | 任务栏按钮样式 |
|----------|----------------|
| 激活 | 凹陷效果（sunken） |
| 非激活 | 凸起效果（raised） |
| 最小化 | 凸起效果，灰色文字 |

### 点击行为

| 操作 | 当前窗口状态 | 结果 |
|------|--------------|------|
| 单击任务栏按钮 | 激活 | 最小化窗口 |
| 单击任务栏按钮 | 非激活 | 激活并置顶窗口 |
| 单击任务栏按钮 | 最小化 | 还原窗口并激活 |

## 边界情况处理

### 1. 多窗口重叠

**问题**: 打开多个窗口后如何管理层级？

**方案**: 
- 使用"最近激活优先"原则
- 每次点击窗口将其提升到顶层
- 最多支持 999 层层级

### 2. 窗口超出屏幕

**问题**: 拖拽导致窗口部分超出屏幕？

**方案**:
- 限制窗口左上角不能超出屏幕
- 右下角允许超出（出现滚动条）
- 提供"适应屏幕"快捷操作

### 3. 快速连续点击

**问题**: 用户快速双击可能导致多次打开？

**方案**:
- 添加 300ms 防抖
- 检查应用是否已打开
- 如已打开则聚焦已有窗口
