# Cabin 项目计划文档

## 1. 项目概述

### 1.1 项目目标
Cabin 项目旨在创建一个 Windows 95 风格的个人网站/博客平台，提供复古桌面操作系统的 Web 交互体验。项目核心目标包括：
- 完整的 Windows 95 桌面环境模拟
- 静态内容展示（博客文章、文档）
- 虚拟文件系统管理
- 完全客户端运行，支持静态导出

### 1.2 项目范围
**包含功能**：
- 桌面环境（图标、任务栏、壁纸）
- 应用程序系统（文件资源管理器、记事本等）
- 虚拟文件系统（预定义目录结构）
- 内容管理（Markdown 博客和文档）
- 静态导出和部署

**不包含功能**：
- 用户认证和登录系统
- 后端 API 服务
- 实时协作功能
- 复杂的数据持久化

### 1.3 成功标准
- **功能验收**：所有核心功能正常工作
- **用户体验**：符合 Windows 95 交互习惯
- **性能指标**：首屏加载时间 < 2s，交互响应时间 < 100ms
- **部署能力**：支持 GitHub Pages 静态部署
- **代码质量**：无 ESLint 错误，构建成功

## 2. 当前状态评估

### 2.1 已完成工作
- ✅ **基础架构**：Next.js App Router + React95 UI 框架
- ✅ **桌面环境**：桌面图标、任务栏、窗口管理系统
- ✅ **应用系统**：文件资源管理器、记事本、博客应用
- ✅ **内容管理**：Markdown 内容自动加载和渲染
- ✅ **虚拟文件系统**：预定义的 Windows 95 目录结构
- ✅ **构建流程**：自动化元数据生成和静态导出

### 2.2 待解决问题（验收阻塞点）
- ❌ **StaticFileSystem 加载问题**：运行时出现 `Cannot read properties of undefined (reading '/')` 错误
- ❌ **桌面图标显示异常**：显示12个空白图标而非6个预设图标
- ❌ **调试日志残留**：控制台存在大量调试输出和 ESLint 警告
- ❌ **路由系统冲突**：需要确保 App Router 正确配置

### 2.3 根本原因分析
1. **模块加载时序问题**：Next.js SSR 和客户端加载时序不一致导致 StaticFileSystem 为 undefined
2. **架构冗余**：同时存在静态配置和动态 VFS，造成复杂性和潜在冲突
3. **开发残留**：调试代码未清理，影响生产环境体验
4. **客户端组件标记**：部分使用 Hooks 的组件未正确标记 "use client"
