# Cabin 项目需求定义文档

## 1. 项目概述

Cabin 是一个模仿 Windows 95 风格的个人网站/博客平台，提供复古桌面操作系统的 Web 交互体验。

### 1.1 核心目标
- 提供完整的 Windows 95 桌面环境模拟
- 支持静态内容展示（博客文章、文档）
- 实现虚拟文件系统管理
- 完全客户端运行，无需后端服务
- 支持静态导出和部署

### 1.2 目标用户
- 喜欢复古设计风格的个人用户
- 开发者作品集展示
- 技术博客作者
- 怀旧 UI 爱好者

## 2. 功能需求

### 2.1 桌面环境
- **桌面图标**：显示预设的6个应用图标
  - 我的电脑 (my-computer)
  - 回收站 (recycle-bin)
  - 我的文档 (my-documents)
  - 我的图片 (my-pictures)
  - 我的博客 (my-blog)
  - 可移动磁盘 (removable)
- **任务栏**：位于底部，包含开始按钮和活动窗口指示器
- **壁纸**：支持自定义背景色
- **右键菜单**：桌面右键显示上下文菜单

### 2.2 应用程序系统
- **文件资源管理器**：浏览虚拟文件系统目录结构
- **记事本**：查看和编辑 文本格式的内容
- **我的电脑**：访问 C: 和 D: 虚拟驱动器
- **我的博客**：专门浏览博客文章
- **我的文档**：查看个人文档
- **可移动磁盘**：支持用户上传本地文件

### 2.3 虚拟文件系统
- **默认内容加载**：从 `content/` 目录自动加载预定义内容，其中包括Markdown文档
- **目录结构**：预定义的 Windows 95 风格目录结构
- **文件元数据**：支持文件类型、图标、应用关联
- **动态扩展**：通过可移动磁盘功能添加用户文件

### 2.4 内容管理
- **博客文章**：存储在 `content/posts/` 目录
- **文档页面**：存储在 `content/pages/` 目录
- **自动构建**：构建时自动生成虚拟文件系统元数据
- **Markdown 支持**：完整的 Markdown 渲染能力

### 2.5 部署要求
- **静态导出**：支持 `next export` 静态生成
- **GitHub Pages 兼容**：适配 GitHub Pages 部署
- **无后端依赖**：完全客户端运行
- **SEO 友好**：支持搜索引擎索引

## 3. 非功能需求

### 3.1 性能要求
- **首屏加载**：静态内容预渲染，快速加载
- **内存占用**：轻量级实现，适合低配置设备
- **响应速度**：桌面交互流畅，无明显延迟

### 3.2 兼容性要求
- **浏览器支持**：现代浏览器（Chrome, Firefox, Safari, Edge）
- **移动端**：基本可用，但主要针对桌面体验
- **屏幕尺寸**：支持常见桌面分辨率

### 3.3 可维护性
- **模块化架构**：清晰的组件分离
- **配置驱动**：通过配置文件管理应用和内容
- **自动化构建**：减少手动维护成本

## 4. 验收标准

### 4.1 基础功能验收
- [ ] 桌面显示6个预设应用图标
- [ ] 开始菜单按钮具有正确的 Windows 95 3D 样式
- [ ] 双击桌面图标能打开对应应用窗口
- [ ] 窗口支持拖拽、最小化、关闭操作
- [ ] 任务栏正确显示活动窗口

### 4.2 内容功能验收
- [ ] 博客文章能正常显示（4篇文章）
- [ ] 文档页面能正常显示（2个文档）
- [ ] 文件资源管理器能浏览完整目录结构
- [ ] 记事本应用能正确渲染 Markdown 内容

### 4.3 部署验收
- [ ] 项目能成功构建 (`npm run build`)
- [ ] 支持静态导出 (`npm run export`)
- [ ] 在本地开发服务器正常运行 (`npm run dev`)
- [ ] 无运行时错误和控制台报错

### 4.4 用户体验验收
- [ ] Windows 95 视觉风格一致
- [ ] 交互符合 Windows 95 操作习惯
- [ ] 按钮和控件具有正确的 3D 效果
- [ ] 界面布局合理，无重叠或错位
