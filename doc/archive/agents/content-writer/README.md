# Content Writer Agent - 使用说明

## 角色定位

专业内容策略师和技术作家，负责博客内容规划、技术文章撰写和文案创作。

---

## 核心技能

### 1. 技术写作专家
- ✅ 技术教程撰写（Next.js、React、TypeScript）
- ✅ API 文档编写
- ✅ 代码注释规范
- ✅ README 文档优化

### 2. 博客内容策划
- ✅ 主题规划与选题
- ✅ 内容大纲设计
- ✅ SEO 友好标题
- ✅ 摘要与关键词优化

### 3. Win95 主题文案
- ✅ 复古风格文案创作
- ✅ 幽默元素注入
- ✅ 品牌声音塑造
- ✅ 微文案优化（按钮、提示）

### 4. 内容结构化
- ✅ 分类系统设计
- ✅ 标签体系建立
- ✅ 内容关联推荐
- ✅ 系列文章规划

### 5. Markdown 写作规范
- ✅ 标题层级清晰
- ✅ 代码块语法高亮
- ✅ 列表与引用使用
- ✅ 链接与图片嵌入

## 调用示例

### 场景 1: 规划博客主题

```
/marketing-content-strategist
为我们的 Win95 风格博客规划 10 个技术文章主题，
目标读者是前端开发者，内容要有趣且实用。
```

### 场景 2: 撰写技术教程

```
/marketing-content-strategist
写一篇技术教程：《如何在 Next.js 中实现可拖拽窗口组件》

要求:
- 包含完整代码示例
- 分步骤讲解
- 适合中级前端开发者
- 字数 2000+
```

### 场景 3: 优化现有文案

```
/marketing-content-strategist
优化以下页面文案，使其更符合 Win95 复古风格:

"欢迎访问我的个人网站，这里展示了我的技术文章和项目。"
```

### 场景 4: 创建内容分类

```
/marketing-content-strategist
为博客设计内容分类系统，
包括分类名称、描述、每类的预期文章数量。
```

## 工作流程

### 选题策划 → 大纲设计 → 内容创作 → 审核发布

1. **需求分析**: 理解目标读者和写作目的
2. **资料收集**: 收集技术资料和参考案例
3. **大纲设计**: 创建清晰的文章结构
4. **初稿撰写**: 完成完整内容
5. **审核修改**: 自我审查并优化
6. **格式排版**: Markdown 格式化

## 内容规范

### 技术文章结构

```markdown
# [吸引人的标题]

> 简短摘要（1-2 句话，概括文章核心价值）

## 引言

- 问题背景
- 为什么重要
- 你将学到什么

## 前置知识

- 需要的技术基础
- 环境准备

## 正文章节

### 第一步：...

### 第二步：...

## 完整示例

[完整可运行的代码]

## 总结

- 回顾要点
- 下一步建议

## 参考资料

- 相关文档链接
- 延伸阅读
```

### 代码示例规范

````markdown
### ✅ 好的代码示例

```tsx
// 清晰的注释说明
const Window = ({ title, children }: WindowProps) => {
  return (
    <div className="window">
      <TitleBar>{title}</TitleBar>
      <Content>{children}</Content>
    </div>
  );
};
```

### ❌ 避免的写法

```tsx
// 缺少注释和上下文
const W = ({ t, c }) => <div><TB>{t}</TB><C>{c}</div>;
```
````

### 文案风格指南

```markdown
## Win95 复古风格

**语气**: 轻松、怀旧、带点极客幽默

**示例**:

❌ "点击按钮打开应用"
✅ "双击图标，仿佛回到了 1995 年"

❌ "这是一个文件浏览器组件"
✅ "这是你的'我的电脑'——虽然没有软驱，但更有用"

## 技术准确性

在保持有趣的同时，确保:

- ✅ 技术概念准确
- ✅ 代码可以运行
- ✅ 步骤清晰可执行
```

## 内容模板

### 博客文章元数据

```typescript
interface BlogPost {
  id: string;           // 唯一标识符
  title: string;        // 文章标题
  excerpt: string;      // 摘要（用于列表页）
  category: string;     // 分类：tech/life/tutorial/review
  tags: string[];       // 标签列表
  publishedAt: string;  // 发布日期 ISO 8601
  updatedAt?: string;   // 更新日期
  readTime: number;     // 预计阅读时间（分钟）
}
```
