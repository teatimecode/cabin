# UX Architect Agent - 使用说明

## 角色定位

专家级 UX 架构师，专注于 Win95 桌面环境的交互设计、用户流程规划和体验优化。

---

## 核心技能

### 1. 窗口交互设计专家
- ✅ 窗口管理器行为设计（拖拽、缩放、层叠）
- ✅ 多窗口状态管理（激活、最小化、最大化）
- ✅ 窗口焦点与 z-index 层级
- ✅ 任务栏窗口预览

### 2. 文件导航系统设计
- ✅ 文件夹层级导航（进入/返回/面包屑）
- ✅ 地址栏路径显示
- ✅ 历史记录前进/后退
- ✅ 快捷方式布局策略

### 3. 菜单与操作流
- ✅ 开始菜单结构规划
- ✅ 右键上下文菜单
- ✅ 菜单栏操作流（文件、编辑、查看、帮助）
- ✅ 键盘快捷键设计

### 4. 用户体验研究
- ✅ 用户操作流程映射
- ✅ 痛点识别与优化
- ✅ 可用性测试计划
- ✅ 用户反馈分析

### 5. 信息架构设计
- ✅ 内容分类系统
- ✅ 导航层次结构
- ✅ 搜索功能设计
- ✅ 书签/收藏系统

## 调用示例

### 场景 1: 设计窗口管理器

```
/design-ux-architect
设计一个完整的窗口管理器交互流程，包括:
- 如何打开新窗口
- 如何切换窗口焦点
- 如何最小化到任务栏
- 如何关闭窗口的动画反馈
```

### 场景 2: 优化文件导航

```
/design-ux-architect
当前的文件夹导航只有进入功能，
请设计完整的导航体验，包括返回上级、面包屑导航、历史记录等。
```

### 场景 3: 设计右键菜单

```
/design-ux-architect
为桌面图标和文件浏览器设计右键上下文菜单，
列出应该包含的操作项及其优先级。
```

### 场景 4: 用户流程映射

```
/design-ux-architect
绘制用户从打开博客文章到阅读完成的完整流程图，
识别可能的摩擦点并提出优化建议。
```

## 工作流程

### 用户研究 → 流程设计 → 原型验证 → 文档输出

1. **理解用户**: 分析目标用户和使用场景
2. **流程 mapping**: 绘制用户操作流程图
3. **交互设计**: 设计每个触点的具体行为
4. **文档输出**: 创建详细的交互说明文档

## 设计规范

### 窗口交互规范

```markdown
## 窗口打开流程

1. 用户双击桌面图标
2. 播放打开动画 (150ms)
3. 窗口出现在桌面中央偏上位置
4. 自动获得焦点并置顶
5. 任务栏对应按钮高亮

## 窗口拖拽规范

- 触发区域：标题栏
- 延迟：按下后 50ms 开始拖拽
- 边界：不能拖出手册区域
- 吸附：接近边缘时自动吸附

## 窗口层级管理

- 新打开窗口：zIndex = 最大值 + 1
- 点击窗口：提升到顶层
- 最小化：从桌面隐藏，保留在任务栏
```

### 导航设计规范

```markdown
## 面包屑导航格式

我的电脑 > 我的博客 > Hello World

- 分隔符：`>` (大于号)
- 可点击：除最后一项外的所有节点
- hover 效果：下划线 + 颜色变化

## 历史记录管理

- 最大历史深度：50 条
- 前进/后退按钮状态：
  - 无历史记录时禁用
  - 到达边界时禁用对应按钮
```

### 快捷键设计

```markdown
## 全局快捷键

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| Ctrl+W | 关闭当前窗口 | 同浏览器行为 |
| Alt+Tab | 切换窗口 | 经典 Windows 行为 |
| F5 | 刷新 | 重新加载内容 |
| Ctrl+O | 打开文件 | 快速打开对话框 |

## 文件浏览器快捷键

| 快捷键 | 功能 |
|--------|------|
| Backspace | 返回上级目录 |
| Enter | 打开选中项 |
| Delete | 删除选中项 |
| Ctrl+A | 全选 |
```

## 用户流程模板

### 流程图示例

```
用户双击"我的博客"图标

    ↓
```

### 状态转换图

```
窗口状态机:
[关闭] --双击图标--> [打开中] --动画完成--> [激活]
[激活] --点击最小化--> [最小化]
[最小化] --点击任务栏--> [激活]
[激活] --点击关闭--> [关闭]
[激活] --点击其他窗口--> [非激活]
[非激活] --点击--> [激活]
```

## 交付物清单

### 1. 交互流程图

使用 Mermaid 或文字描述用户操作流程

### 2. 状态管理文档

```markdown
## 窗口状态

interface WindowState {
  id: string;
  isOpen: boolean;
  isMinimized: boolean;
  isMaximized: boolean;
  isActive: boolean;
  position: { x, y };
  size: { width, height };
}
```

### 3. 交互说明文档

```markdown
## 窗口拖拽交互

**触发条件**: 鼠标在标题栏按下并移动超过 5px

**拖拽过程**:
1. 创建半透明跟随层
2. 实时更新窗口位置
3. 显示边界吸附提示

**释放行为**:
- 在有效区域内：更新最终位置
- 超出边界：回弹到合法位置
```
